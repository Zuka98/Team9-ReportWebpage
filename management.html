Management

Legal Issues

Potential Liabilities

  • Unauthorised access to data subject’s data through direct sharing of data from the application, e.g. by data subject’s own error or a bug that may trigger a data share without consent

  • Data subject may be uniquely identified if they are the sole user of the application within their postcode area

  • Android SDK is subject to US export laws and regulations which could have imposed liabilities if application built using that SDK was shipped to a region where the USA restricts or regulates the export of intellectual property. However, our application is created for NHS digital which only intends to use it for people living in the United Kingdom, therefore this application should not be the subject to breaching above mentioned regulations

  • In the case of manipulating user data in an unanticipated way, unauthorized usage or access to any information will raise liabilities to European Parliament and Council of the European Union, Google and other parties whose data was stored and used by the application. However, this is not a possibility for this project as score prediction using machine learning model happens locally on the device and not on the cloud. Every single piece of recorded information never leaves user's phone. Only limited, anonymous and randomized data is shared with the real-time nationwide database if and only if the user disables privacy filter. This implies that by doing so user accepts the terms and conditions of the application about which he/she gets full, symmetric information and these terms doesn’t


Intellectual Property

  • This project is developed for Android devices; therefore, Android Studio was chosen as a development IDE, which is available for open source use according to Android Software Development Kit License Agreement. Regarding the use of other Android libraries and API's, it is restricted to use data that might be protected by intellectual property rights, however, in this project, no such information or data was imported and used.

  • The types of intellectual properties that protect the app are copyrights and trademarks. As the app would be deployed as an NHS app, the NHS logo would be used, which is both trademarked and copyrighted. Additionally, the logo and name of the app, WellWellWell, will need to be trademarked to prevent unauthorised usage of the name. It is unclear if the “5 Ways to Well-being” is copyrighted or trademarked. It was produced by Foresight, the Government Ofﬁce for Science, and as such is likely trademarked. The NHS is authorised to use the name and methods.


Privacy

  • One important addition to this application we implemented was the inclusion of a Terms and Conditions and a Data Protection Statement that is shown to the user upon initial installation and setup that the user is required to agree to prior to using the application. The application will generate a predicted “well-being score”, which is initially only shared with the user. The purpose of this application is to provide GPs and healthcare professionals nationwide to better understand individuals and the nation as a whole better, so a fundamental aspect of the application involves data sharing with these parties. We have implemented “Button A” and “Button B”, as described in the Liability section above. The information collected and shared using “Button A” include the data subject’s name, postcode (location), well-being sore and the breakdown of their well-being data in a visualised format. The information shared using “Button B” only include the data subject’s postcode and well-being score. There is a concern about the anonymity of data subjects on the nationwide database as they may be uniquely identiﬁed using their postcode. We were asked to combat this issue by randomising 30% of the data, i.e. there is a 30% chance that the well-being score saved on the nationwide database is a randomly generated number so no one particular datapoint can be 100% accurate and attributable to any one data subject.
