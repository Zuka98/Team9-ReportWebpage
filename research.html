Research

Details of our research into existing and applicable technologies and further information that may benefit the project.

Existing technologies

This project is relevant and paramount. Which is why we were surprised to realise there were limited, if any, such existing projects or technologies. There are an abundance of mental health related apps, however, from which inspiration can be drawn over how data is collected and used in their methods.

Related apps

What's Up?

What's Up gives the user an insight into their thinking patterns through a diary using two different therapy methods. It also provides information about keeping relaxed through breathing techniques, quotes for inspiration and a discussion forum.

Moodpath

Moodpath is designed a "pre-program" for a professional consultation. Through a 2 week program where the user answers some questions 3 times a day, the app produces an assessment to be brought to a consultation.

WellMind

This is an NHS app that gets users to track their emotions and feelings, set goals to try to complete and reminders to work towards them. It also provides useful information, audio tracks for relation and phone numbers to contact within the NHS and out for help.

Woebot

Woebot is a chatbot that mimics the experience of talking to a therapist. By using CBT-style feedback and mood tracking, the AI monitors the users' emotions and mood and offers techniques and tips based on the conversation.

Difficulties encountered

The biggest problem we encountered was moving the app from Android to iOS. This was primarly due to two reasons. Firstly, the original design being coded without consideration of cross platform development, which meant a complete rewrite of code would have been necessary to redevelop the application (in a very limited timeframe). Secondly, iOS has limited number of services that are allowed to run in the background. This made tracking data in realtime difficult, which is a vital element of the app. While this difficulty can probably be overcome by an experienced iOS developer, it has become a difficulty we are not able to overcome with the timeframe, and as such have decided to abandon the idea of an iOS app in this development cycle.

Research Topics

Natural Language Processing (NLP)

Natural Language Processing is a subfield of artificial intelligence focusing on the analysis and processing of human or natural languages. This includes detecting usage of particular words, phrases and sentence structures to name a few. Sentiment analysis was the most relevant and appropriate method to use for our purposes, which breaks down sentences into sentiments and classifies their type based on their location in a sentence. This was to be used for analysing a diary feature as part of the application to determine a person’s mental well-being based on their writing style. It was, however, abandoned and as such this research was not used.

Data Randomisation for Anonymity

Due to GDPR regulations, we could not allow any one particular person to be uniquely identifiable with information that is meant to be anonymous. As such, our method of randomisation allowed for anonymity while maintaining strong data integrity. 70% of all data is accurate, whereas 30% has been randomised to ensure that no one particular data subject can be identified and attributed to the queried score, even if they are the only individual in the identified postcode.

5 Ways to Wellbeing

The 5 Ways to Wellbeing is an NHS approved framework for gauging a person's mental wellbeing. The 5 ways are

  • Connect - How actively engaged you are in communication with people
  • Be Active - How physically active you are, walking, cycling, etc.
  • Keep Learning - How interested you are in learning new skills and things
  • Give to Others - How charitable you are, relating to a sense of community
  • Be Mindful - How aware you are of your surroundings, including your thoughts and feelings

Bayes' Theorem

Bayes' theorem is a statistical theorem that describes probabilities given that a previous event has occured. Bayes' theorem is described using conditional probability and uses the probabilities of previous events that have occured to predict the probability of another event occuring. This is particularly useful in predicting a wellbeing score given that the user can report an error in prediction, either positive or negative. Using the theorem, it can take in user feedback and determine the likely accurate score for the user and population.

Whole networked web system and visualisation

We have developed a visualisation system that allows the visualisation of any set of data stored on the cloud based on postcode area in the United Kingdom. While it was designed for use in the UK, the way we designed the framework should allow it to be repurposed for use in any country and region.

Background processes in Android and iOS

In our research, we found that iOS does not allow applications to run background processes for a sufficient amount of time for the WellWellWell application to be practically deployed on the operating system and as such, have decided to only develop the application on Android. Additionally, iOS does not allow applications to query application usage, which is necessary for tracking the number of minutes on call, texts, etc. The Android application uses the pedometer and queries application usage for data collection.
